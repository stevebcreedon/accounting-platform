# Phase 1 — UI Design Contract

> Visual and interaction contract for the project scaffold. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> Phase 1 establishes the design token foundation that every subsequent phase inherits. There are no interactive components in this phase -- only the root layout, global styles, and a branded placeholder page. The contract below defines the visual system that all future components must conform to.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none (custom Tailwind tokens only) |
| Icon library | none (deferred to Phase 3+) |
| Heading font | Satoshi (variable, loaded via next/font/local from Fontshare woff2 files) |
| Body font | DM Sans (variable, loaded via next/font/google) |

**Primary focal point:** The brand wordmark "The Ledger" rendered in Satoshi at hero scale, centered on the cream background -- the single visual anchor on the Phase 1 placeholder page.

**Source:** CONTEXT.md D-05, D-06; RESEARCH.md Pattern 1

**Why no shadcn:** The project uses a bespoke warm editorial design language (Monocle/Stripe Press references) that does not align with shadcn's neutral base. Custom Tailwind tokens provide full control over the distinctive brand identity. Component primitives may be introduced in later phases if needed.

## Spacing Scale

Declared values (all multiples of 4):

| Token | Value | Tailwind Class | Usage |
|-------|-------|----------------|-------|
| xs | 4px | `p-1` / `gap-1` | Icon gaps, inline padding |
| sm | 8px | `p-2` / `gap-2` | Compact element spacing, pill padding |
| md | 16px | `p-4` / `gap-4` | Default element spacing, card padding |
| lg | 24px | `p-6` / `gap-6` | Section padding, content gutters |
| xl | 32px | `p-8` / `gap-8` | Layout gaps between major elements |
| 2xl | 48px | `p-12` / `gap-12` | Major section breaks |
| 3xl (editorial override) | 72px | `p-18` / custom `py-18` | Hero vertical padding, page-level breathing room |
| 4xl (editorial override) | 120px | `py-30` / custom `py-30` | Top-of-page hero vertical spacing |

Exceptions:
- **Editorial overrides:** Custom spacing tokens `18` (4.5rem / 72px) and `30` (7.5rem / 120px) are added to tailwind.config.ts. These exceed the standard 8-point scale and are justified by the generous editorial whitespace requirement (D-08). They are used exclusively for hero-level vertical spacing and page-level breathing room -- not for component-level layout.
- Content gutters: `px-6` (24px) on mobile, `px-8` (32px) on tablet+.

**Source:** CONTEXT.md D-08 (generous whitespace); RESEARCH.md Pattern 3

## Typography

### Type Scale (4 sizes)

Size variation creates hierarchy. Weight variation differentiates roles within the same size.

| Token | Size | Weight | Line Height | Letter Spacing | Font Family | Usage |
|-------|------|--------|-------------|----------------|-------------|-------|
| `text-lg` | 32px (2rem) | 700 | 1.15 | -0.02em | Satoshi | H1, hero headings, display text. Use 700 for page titles, 400 for pull quotes. |
| `text-md` | 24px (1.5rem) | 700 | 1.25 | -0.01em | Satoshi | H2, H3 headings, section titles. Use 700 for H2, 400 for H3 subheadings. |
| `text-base` | 16px (1rem) | 400 | 1.7 | 0 | DM Sans | Body text, descriptions, article content. Use 700 for inline bold/emphasis. |
| `text-sm` | 14px (0.875rem) | 400 | 1.6 | 0 | DM Sans | Meta text, captions, timestamps, labels. Use 700 for label emphasis. |

### Responsive Scale for Hero/Display

Hero and display sizes are NOT separate tokens. They are responsive variants of `text-lg` applied via Tailwind responsive prefixes:

| Context | Mobile | Tablet (md:) | Desktop (lg:) |
|---------|--------|--------------|---------------|
| Hero heading | 32px (`text-lg`) | 48px (`md:text-[3rem]`) | 56px (`lg:text-[3.5rem]`) |
| Page H1 | 32px (`text-lg`) | 40px (`md:text-[2.5rem]`) | 40px (`lg:text-[2.5rem]`) |

These responsive utilities use arbitrary values, not additional tokens.

### Font Weights Used (2 weights)

| Weight | Value | Usage |
|--------|-------|-------|
| Regular | 400 | Body text, descriptions, meta information, H3 subheadings, pull quotes |
| Bold | 700 | H1, H2 headings, brand wordmark, inline emphasis, label emphasis |

### Prose Typography (Article Content)

**Source:** CONTEXT.md D-05 through D-07; RESEARCH.md Patterns 3 and 4

### Accent Reserved For

The burnt-orange accent (#E8720C) is reserved exclusively for:

1. **Hyperlinks** in prose/article content
2. **Focus-visible outlines** (2px solid, 2px offset) on all interactive elements
3. **Blockquote left borders** in prose content
4. **Text selection background** (at 15% opacity)
5. **Primary CTA buttons** (future phases -- not present in Phase 1)

Accent is NOT used for: headings, body text, background fills, borders, or decorative elements.

### Semantic Colors (Utility)

| Success | #16A34A (green-600) | Confirmation states (future phases) |
| Warning | #CA8A04 (yellow-600) | Warning notices (future phases) |

## Copywriting Contract

Phase 1 is a scaffold with a single placeholder landing page. Copywriting is minimal.

| Element | Copy |
|---------|------|
| Page title (metadata) | "The Ledger -- Irish Accounting Guidance" |
| Title template | "%s \| The Ledger" |
| Meta description | "Clear, jargon-free accounting guidance for Irish small businesses, sole traders, and company directors." |
| Hero heading (placeholder page) | "The Ledger" |
| Hero body (placeholder page) | "Clear, jargon-free accounting guidance for Irish small businesses." |
| Empty state heading | Not applicable -- Phase 1 has no data-driven UI |
| Empty state body | Not applicable |
| Error state | Not applicable -- Phase 1 has no user interactions |
| Destructive confirmation | Not applicable -- Phase 1 has no destructive actions |

**Brand voice notes for downstream phases:**
- Tone: Authoritative but approachable. "Your knowledgeable friend who happens to be an accountant."
- Never attribute to a personal author. The brand is the authority (per CLAUDE.md).
- Ireland-specific: Always say "Revenue" not "HMRC", "CRO" not "Companies House", "euro" not "pounds".
- Jargon-free: If a term requires explanation, explain it inline or avoid it entirely.

**Source:** CONTEXT.md D-11; RESEARCH.md Code Examples; CLAUDE.md constraints

## Elevation & Card Styles

| Token | Shadow Value | Usage |
|-------|-------------|-------|
| `shadow-card` | `0 1px 3px 0 rgba(0,0,0,0.04), 0 1px 2px -1px rgba(0,0,0,0.04)` | Card resting state |
| `shadow-card-hover` | `0 10px 15px -3px rgba(0,0,0,0.06), 0 4px 6px -4px rgba(0,0,0,0.04)` | Card hover state |

Cards are white (#FFFFFF) on cream (#FAF8F5) background with `shadow-card`. On hover, transition to `shadow-card-hover` with a subtle lift (translate-y, to be animated in Phase 9). Rounded corners: `rounded-xl` (12px).

**Source:** CONTEXT.md D-10; RESEARCH.md Pattern 3
