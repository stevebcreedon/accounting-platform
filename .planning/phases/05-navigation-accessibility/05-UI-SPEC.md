# Phase 5 — UI Design Contract

> Visual and interaction contract for navigation and accessibility. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> Phase 5 wraps all existing content in a complete responsive layout shell: sticky header with "The Ledger" wordmark and 3 nav links, 3-column footer with disclaimer, full-screen mobile menu overlay, skip-to-content link, WCAG 2.1 AA compliance across all pages, and print-friendly article styles. All design tokens, typography, color, and spacing are inherited from Phase 1. No new design tokens are introduced.

Exceptions: none -- all values are multiples of 4 from the Phase 1 scale.

### Accent Usage in Phase 5

The burnt-orange accent (#E8720C) is used exclusively for:

1. **Active nav link indicator** -- `border-b-2 border-burnt-orange-500` on the current page's nav link (desktop)
2. **Nav link hover state** -- text color shifts to burnt-orange-500
3. **Mobile overlay active link** -- burnt-orange-500 text color on the current page's link
4. **Focus-visible outlines** on all interactive elements (inherited from Phase 1)
5. **Skip-to-content link background** -- burnt-orange-500 bg with white text when focused

Accent is NOT used for: header background, footer background, hamburger icon, footer links (resting), mobile overlay background, print styles.

## Accessibility Requirements (WCAG 2.1 AA)

This phase establishes site-wide accessibility compliance. All requirements below apply across every page.

### Semantic HTML

| Requirement | Implementation |
|-------------|----------------|
| Landmark regions | `<header>`, `<nav>`, `<main>`, `<footer>` on every page |
| Skip link | First focusable element, targets `#main-content` |
| Heading hierarchy | Single `<h1>` per page, no heading level skips (H1 > H2 > H3) |
| Lists | Navigation uses `<ul>` / `<li>` for link lists |
| Buttons vs links | Hamburger/close are `<button>`. Nav items are `<a>` / `<Link>`. |

### Keyboard Navigation

**Contrast notes:**
- Burnt-orange-500 as text on cream does NOT meet 4.5:1 for normal text. - Stone-500 is used only for tertiary meta text (copyright, print header) where reduced emphasis is intentional and text is non-essential.
