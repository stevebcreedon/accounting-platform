Per D-09, D-10:

1. Update vercel.json to add a daily rebuild cron at 06:00 UTC alongside the existing analytics rollup cron. The rebuild cron should call a new API route `/api/cron/rebuild` that triggers on-demand ISR revalidation.

Create the route at `src/app/api/cron/rebuild/route.ts`:

```typescript
import { revalidatePath } from 'next/cache';
import { NextRequest, NextResponse } from 'next/server';

export async function GET(request: NextRequest) {
  const authHeader = request.headers.get('authorization');
  if (authHeader !== `Bearer ${process.env.CRON_SECRET}`) {
    return NextResponse.json({ error: 'Unauthorized' }, { status: 401 });
  }
  
  revalidatePath('/', 'layout');
  return NextResponse.json({ revalidated: true, now: Date.now() });
}
```

Update vercel.json to include:
```json
{
  "crons": [
    { "path": "/api/cron/rollup", "schedule": "0 3 * * *" },
    { "path": "/api/cron/rebuild", "schedule": "0 6 * * *" }
  ]
}
```

NOTE: For static content built with Velite, the actual rebuild happens on deploy, not via ISR revalidation. The better approach for scheduled publishing is a Vercel Deploy Hook triggered by a cron. However, Vercel crons can only call API routes in the same project. The simplest working approach: the cron route triggers revalidation of dynamic paths, which works for ISR-enabled pages. For fully static (SSG) pages, a GitHub Action or Vercel Deploy Hook is needed for a full rebuild. Add a comment in the route explaining this.

2. Run `npm run build` locally to verify all 72 articles compile successfully through the Velite pipeline. The build must complete without errors.

3. Count articles to verify: `ls content/guides/*.mdx | wc -l` should return 72.
