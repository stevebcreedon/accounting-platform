# Phase 3 — UI Design Contract

> Visual and interaction contract for the article page template. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> Phase 3 builds the complete article reading experience on top of Phase 1's design tokens. This phase introduces the first interactive components (FAQ accordion, sticky TOC with scroll tracking) and the first multi-section page layout. All spacing, typography, and color tokens are inherited from Phase 1 UI-SPEC. This contract defines the visual rules for 7 new components and their responsive behavior.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none (custom Tailwind components) |
| Icon library | Lucide React (chevron-down for FAQ, chevron-right for breadcrumbs, clock for reading time, calendar for dates, shield-check for verified badge) |
| Heading font | Satoshi (inherited from Phase 1) |
| Body font | DM Sans (inherited from Phase 1) |

**Primary focal point:** The article title rendered in Satoshi at `text-lg` (32px) on mobile, scaling to 40px via `md:text-[2.5rem]` on tablet+ -- the reader's first anchor before scrolling into the content.

**Icon library rationale:** Lucide is the standard feather-style icon set for Next.js projects. It provides tree-shakeable SVG components. Only 5 icons are needed for this phase -- no full icon library overhead.

**Source:** Phase 1 UI-SPEC (inherited); CONTEXT.md D-03, D-10

## Spacing Scale

Inherited from Phase 1. No new tokens introduced. Phase 3 uses the existing scale:

| Token | Value | Tailwind Class | Phase 3 Usage |
|-------|-------|----------------|---------------|
| xs | 4px | `gap-1` | Icon-to-label gaps in meta line items |
| sm | 8px | `gap-2` / `p-2` | Breadcrumb separator spacing, TOC link padding |
| md | 16px | `p-4` / `gap-4` | FAQ item padding, key takeaways list padding, related card internal padding |
| lg | 24px | `p-6` / `gap-6` | Key takeaways box padding, related articles card padding, TOC sidebar padding |
| xl | 32px | `gap-8` / `mb-8` | Spacing between major article sections (meta to TOC, body to key takeaways) |
| 2xl | 48px | `py-12` / `mb-12` | Top/bottom page padding, spacing before email CTA |
| 3xl | 72px | `py-18` | Not used in Phase 3 (hero-level only) |

### Section Vertical Rhythm

Every top-level article section is separated by `mb-12` (48px). Within sections, elements use `mb-8` (32px). This creates a clear two-tier rhythm: major breaks between sections, moderate breaks within.

| Section Boundary | Spacing | Class |
|-----------------|---------|-------|
| Breadcrumbs to title | 16px | `mb-4` |
| Title to meta line | 16px | `mb-4` |
| Meta line to intro paragraph | 32px | `mb-8` |
| Intro to article body | 32px | `mb-8` |
| Article body to key takeaways | 48px | `mb-12` |
| Key takeaways to FAQ | 48px | `mb-12` |
| FAQ to email CTA | 48px | `mb-12` |
| Email CTA to related articles | 48px | `mb-12` |
| Page top padding | 48px | `pt-12` |
| Page bottom padding | 48px | `pb-12` |

Exceptions: none -- all values are multiples of 4 from the Phase 1 scale.

**Source:** Phase 1 UI-SPEC spacing scale; CONTEXT.md Claude's Discretion (section spacing)

The burnt-orange accent (#E8720C) is used exclusively for:

1. **TOC active section indicator** -- 2px left border on the active link
2. **Category pill background** -- burnt-orange-50 (#FFF7ED) background with burnt-orange-600 text

6. **Focus-visible outlines** on all interactive elements (inherited from Phase 1)

Accent is NOT used for: FAQ chevrons (use stone-600), breadcrumb separators (use stone-400), TOC inactive links (use stone-600), related article card borders (use stone-200).

## Layout

### Article Page Grid (Desktop: >=1024px)

```
|  gutter  |  breadcrumbs (full width, max-w-article)             |  gutter  |
|  gutter  |  title + meta (max-w-article)  |  TOC sidebar (240px) |  gutter  |
|  gutter  |  intro (max-w-article)          |  TOC sidebar (sticky)|  gutter  |
|  gutter  |  body (max-w-article)           |  TOC sidebar (sticky)|  gutter  |
|  gutter  |  key takeaways (max-w-article)  |                      |  gutter  |
|  gutter  |  FAQ (max-w-article)            |                      |  gutter  |
|  gutter  |  email CTA (max-w-article)      |                      |  gutter  |
|  gutter  |  related articles (full bleed, max-w-7xl)             |  gutter  |
```

### TOC Sidebar Specifications

| Property | Value |
|----------|-------|
| Width | 240px fixed |
| Position | sticky, `top-24` (96px from viewport top) |
| Left margin from article body | 48px (`ml-12`) |
| Breakpoint | Visible at `lg:` (>=1024px); hidden below |
| Mobile fallback | Inline TOC block between intro and body |
| Max height | `calc(100vh - 128px)` with `overflow-y: auto` |

### Content Width Constraints

| Element | Max Width | Class |
|---------|-----------|-------|
| Article body + all sections | 48rem (768px) | `max-w-article` |
| Page container (with TOC) | 80rem (1280px) | `max-w-7xl` |
| Related articles grid | 80rem (1280px) | `max-w-7xl` |
| Content gutter (mobile) | 24px | `px-6` |
| Content gutter (desktop) | 32px | `px-8` |

### Mobile Layout (<1024px)

On viewports below 1024px, the TOC sidebar collapses. The page becomes a single-column layout:

1. Breadcrumbs
2. Title + meta line
3. Intro paragraph
4. **Inline TOC** (collapsed by default, expandable tap target)
5. Article body
6. Key takeaways
7. FAQ
8. Email CTA
9. Related articles

The inline mobile TOC uses a bordered container with "In this article" heading and a chevron-down toggle. When expanded, it shows the full heading list.

**Source:** CONTEXT.md D-01, D-03; Claude's Discretion (TOC sidebar width, breakpoint, mobile adaptations)

## Component Specifications

### 1. Breadcrumbs

**Path:** `Home > Guides > {Category Name} > {Article Title}`

| Property | Value |
|----------|-------|
| Separator | Lucide `chevron-right` at 14px, stone-400 color |
| Link color | stone-600 |
| Link hover | burnt-orange-500 |
| Current page (last item) | stone-600, no link, truncated with `max-w-[200px]` and ellipsis on mobile |
| Font | text-sm (14px), weight 400 |
| Spacing between items | 8px (`gap-2`) |
| Vertical margin | `mb-4` below breadcrumbs |

**Semantic HTML:** `<nav aria-label="Breadcrumb">` with `<ol>` list. Last item uses `aria-current="page"`.

### 2. Meta Line

Displays article metadata in a horizontal row with divider dots.

| Item | Icon | Format | Condition |
|------|------|--------|-----------|
| Published date | Lucide `calendar` 14px | "3 Apr 2026" (DD MMM YYYY) | Always shown |
| Updated date | none | "Updated 3 Apr 2026" | Only when `updatedDate` exists |
| Reading time | Lucide `clock` 14px | "8 min read" | Always shown |
| Category pill | none | Category name text | Always shown, links to `/guides/category/{slug}` |
| Verified badge | Lucide `shield-check` 14px | "Verified for 2025/26 tax year" | Only when `updatedDate` exists |

| Property | Value |
|----------|-------|
| Font | text-sm (14px), weight 400, DM Sans |
| Text color | stone-600 |
| Icon color | stone-500 |
| Divider | centered dot (middle dot character), stone-400, `mx-2` spacing |
| Category pill | burnt-orange-50 bg, burnt-orange-600 text, `px-2 py-1`, `rounded-full` |
| Verified badge | green-50 bg, green-700 text, `px-2 py-1`, `rounded-full` |
| Wrap behavior | Wraps naturally on mobile; items are inline-flex |

**Source:** CONTEXT.md D-07, D-08, D-09

### 3. Table of Contents (Sticky Sidebar)

| Property | Desktop (>=1024px) | Mobile (<1024px) |
|----------|---------------------|-------------------|
| Container | White bg, `rounded-lg`, `p-6` | White bg, `rounded-lg`, `p-4`, `border border-stone-200` |
| Heading | "ON THIS PAGE" -- text-sm, weight 700, stone-500, uppercase, `tracking-wider`, `mb-4` | "In this article" -- text-sm, weight 700, charcoal |
| Link list | Vertical list, `space-y-1` | Vertical list, `space-y-1`, toggle visibility |
| Link style | text-sm, stone-600, `py-1 pl-3`, `border-l-2 border-transparent` | text-sm, stone-600, `py-1` |
| Active link | text-sm weight 700, charcoal, `border-l-2 border-burnt-orange-500` | text-sm weight 700, charcoal |
| Hover link | charcoal text color | charcoal text color |
| Transition | `transition-colors duration-150` on all links | none needed |
| Toggle | not applicable | Chevron-down icon rotates 180deg when expanded |

**Active tracking:** Intersection Observer watches all H2 elements. The H2 nearest the top of the viewport (with a rootMargin of `-96px 0px -66%`) marks its corresponding TOC link as active.

**Source:** CONTEXT.md D-01, D-05; Claude's Discretion (TOC width, styling)

### 4. Key Takeaways Box

An MDX component: `<KeyTakeaways>` wrapping a bullet list.

| Property | Value |
|----------|-------|
| Container | burnt-orange-50 bg, 4px burnt-orange-200 left border, `rounded-r-lg`, `p-6` |
| Heading | "Key Takeaways" -- text-md (24px), Satoshi, weight 700, charcoal, `mb-4` |
| List style | Disc bullets in burnt-orange-500, body text (16px), stone-700 |
| List spacing | `space-y-2` between items |
| Max width | Full width within `max-w-article` container |
| Placement | After article body, before FAQ section |

### 5. FAQ Section (Accordion)

An MDX component: `<FAQSection>` containing `<FAQItem question="...">` children.

| Property | Value |
|----------|-------|
| Container | White bg, `rounded-lg`, `overflow-hidden` |
| Section heading | "Frequently Asked Questions" -- text-md (24px), Satoshi, weight 700, charcoal, `mb-6` |
| Question row | `py-4 px-6`, charcoal text, weight 700, `cursor-pointer`, full-width flex between text and chevron |
| Question hover | cream (#FAF8F5) background |
| Chevron icon | Lucide `chevron-down` 20px, stone-500, rotates 180deg when open |
| Chevron transition | `transition-transform duration-200` |
| Answer panel | `px-6 pb-4`, stone-700 text, weight 400 |
| Answer reveal | CSS `grid-rows` transition: `grid-template-rows: 0fr` to `1fr` over 200ms, `overflow: hidden` |
| Divider | `border-b border-stone-200` between items (not after last item) |
| Behavior | One-open-at-a-time -- opening a question closes any other open question |
| Accessibility | `<button>` trigger with `aria-expanded`, `<div>` panel with `aria-hidden` and `role="region"` |

**Animation approach:** Pure CSS grid-row animation -- no motion library needed for this phase. Motion library deferred to Phase 9.

**JSON-LD:** The FAQSection component outputs a `<script type="application/ld+json">` block with FAQPage schema. Each `<FAQItem>` maps to a Question/Answer pair.

**Source:** CONTEXT.md D-04, D-06; Claude's Discretion (FAQ animation)

### 6. Related Articles

| Property | Value |

| Reading time | text-sm (14px), stone-500, `mt-3` |
| Link | Entire card is clickable (`<a>` wrapper) |
| Empty state | If no relatedSlugs or slugs not found, section is not rendered |

**Source:** CONTEXT.md D-05 notes; tailwind.config.ts shadow-card tokens

## Copywriting Contract

| Element | Copy |
|---------|------|
| TOC heading (desktop) | "ON THIS PAGE" |
| TOC heading (mobile) | "In this article" |
| Key takeaways heading | "Key Takeaways" |
| FAQ section heading | "Frequently Asked Questions" |
| Related articles heading | "Related Articles" |
| Email CTA heading | "Stay Updated" |
| Email CTA body | "Email capture form coming soon." |
| Verified badge format | "Verified for {YYYY/YY} tax year" (e.g., "Verified for 2025/26 tax year") |
| Reading time format | "{N} min read" |
| Date format | "D MMM YYYY" (e.g., "3 Apr 2026") |
| Updated date format | "Updated D MMM YYYY" |
| Breadcrumb home | "Home" |
| Breadcrumb guides | "Guides" |
| Pillar badge | "Comprehensive Guide" |
| Disclaimer (bottom of article body) | "This article is for educational purposes only and does not constitute professional accounting advice. For guidance specific to your situation, consult a qualified Irish accountant." |

**Brand voice applied:**
- No personal author attribution. The brand is the authority.
- Ireland-specific: "Revenue" not "HMRC", "CRO" not "Companies House".
- Tax year references use the Irish format (January - December for corporation tax, calendar year for most individuals).
- Dates use European format (day before month).

Inherited from Phase 1. Applied in Phase 3:

| Token | Shadow Value | Phase 3 Usage |
|-------|-------------|---------------|
| `shadow-card` | `0 1px 3px 0 rgba(0,0,0,0.04), 0 1px 2px -1px rgba(0,0,0,0.04)` | Related article cards resting state |

## Accessibility Requirements

| Requirement | Implementation |
|-------------|----------------|
| FAQ accordion | `<button>` triggers with `aria-expanded="true/false"`, answer panels with `role="region"` and `aria-hidden` |
| TOC navigation | `<nav aria-label="Table of Contents">` with `<ol>` list |
| Breadcrumbs | `<nav aria-label="Breadcrumb">` with `<ol>`, last item `aria-current="page"` |
| Skip link | Not in this phase (Phase 5), but article sections use proper heading hierarchy for screen reader navigation |
| Focus indicators | All interactive elements use inherited 2px burnt-orange-500 focus-visible outline |
| Heading hierarchy | H1 (title) > H2 (article sections) > H3 (subsections). No heading level skips. |
| Color contrast | All text/background combinations meet WCAG AA 4.5:1 minimum |
| Keyboard | FAQ buttons operable via Enter/Space. TOC links operable via Tab + Enter. |
