# Phase 9 — UI Design Contract

> Visual and interaction contract for the animation and performance polish layer. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> Phase 9 adds motion to a fully built site. Every layout, component, and page already exists. This phase wraps existing elements in animation primitives without changing their visual design. The contract below defines exact timing, easing, and interaction values so the executor has zero ambiguity.

## Animation Contract

This is the core contract for Phase 9. All values are prescriptive.

### Motion Provider

| Property | Value |
|----------|-------|
| Provider | `LazyMotion` wrapping `{children}` in root layout |
| Features | `domAnimation` (NOT `domMax`) |
| Loading | Dynamic import: `const features = () => import("motion/dom-animation").then(m => m.default)` |
| Boundary | `LazyMotion` is the only `"use client"` boundary at layout level |

**Source:** CONTEXT.md D-08; CLAUDE.md (`motion` ^12)

### Page Transitions (ANIM-01)

| Property | Value |
|----------|-------|
| Type | Opacity fade |
| Initial | `opacity: 0` |
| Animate | `opacity: 1` |
| Exit | `opacity: 0` |
| Duration | 250ms (0.25s) |
| Easing | `easeOut` ([0.0, 0.0, 0.2, 1]) |
| Component | `PageTransition` wrapper in a template.tsx or layout child |
| Reduced motion | Instant (duration: 0) |

**Source:** CONTEXT.md D-04

### Scroll-Triggered Reveals (ANIM-02)

| Property | Value |
|----------|-------|
| Component | `ScrollReveal` wrapper component |
| Initial state | `opacity: 0, y: 16px` (16px upward slide) |
| Animate state | `opacity: 1, y: 0` |
| Duration | 400ms (0.4s) |
| Easing | `easeOut` |
| Trigger | `whileInView` with `viewport: { once: true, amount: 0.2 }` |
| Stagger delay | 100ms between sibling elements (0.1s) |
| Reduced motion | `opacity: 0` to `opacity: 1` only (no y translation), duration 200ms |

**Elements that receive scroll reveals:**

| Page | Elements |
|------|----------|
| Homepage | Hero heading + subtext, Latest Articles section heading + card grid, Category Grid section heading + cards, Email CTA section |
| Article page | Key Takeaways box, Related Articles section |
| Guides hub | Section heading, article grid |
| Category archive | Section heading, article grid |

**Source:** CONTEXT.md D-01 (subtle editorial, 200-400ms, 10-20px slide); CONTEXT.md specifics (one-time, no re-trigger)

### Hover Micro-Interactions (ANIM-03)

#### Article Cards

| Property | Value |
|----------|-------|
| Existing | `shadow-card` to `shadow-card-hover` (already CSS transition-shadow 200ms) |
| Add: Scale | `scale: 1.02` on hover |
| Add: Translate | `translateY: -4px` on hover |
| Duration | 200ms |
| Easing | `easeOut` |
| Implementation | CSS classes added to existing article-card.tsx (no motion library needed) |
| Reduced motion | Shadow transition only (no scale, no translate) |

Tailwind classes to add: `hover:scale-[1.02] hover:-translate-y-1 transition-all duration-200`

**Source:** CONTEXT.md specifics (shadow elevation + slight scale 1.02x)

#### Category Cards

Same treatment as Article Cards.

#### Buttons (CTA buttons, email submit)

| Property | Value |
|----------|-------|
| Scale | `scale: 0.98` on active/pressed |
| Background | Darken to burnt-orange-600 on hover (already exists) |
| Duration | 150ms |
| Easing | `easeInOut` |
| Implementation | CSS `active:scale-[0.98] transition-transform duration-150` |
| Reduced motion | No scale, color transition only |

#### Navigation Links

| Property | Value |
|----------|-------|
| Effect | Underline slides in from left on hover |
| Implementation | CSS `after` pseudo-element, `scaleX(0)` to `scaleX(1)`, `origin-left` |
| Color | burnt-orange-500 (#E8720C) |
| Height | 4px |
| Duration | 200ms |
| Easing | `easeOut` |
| Reduced motion | Instant underline appearance (no slide) |

### FAQ Accordion (ANIM-06)

| Property | Value |
|----------|-------|
| Treatment | **No change** -- keep existing CSS `grid-rows-[0fr]`/`grid-rows-[1fr]` animation |
| Reason | CSS animation already works smoothly; adding motion library increases bundle for no visible gain |

**Source:** CONTEXT.md D-02

### Reading Progress Bar (ANIM-05)

| Property | Value |
|----------|-------|
| Position | Fixed, top of viewport, directly below sticky header (top: 64px / top: 4rem) |
| Width | 100% of viewport |
| Height | 4px |
| Color | burnt-orange-500 (#E8720C) |
| Background | transparent |
| Z-index | 49 (one below header's z-50) |
| Fill direction | Left to right, width = scroll percentage of article content |
| Opacity | 0 when scroll position is 0; fades to 1 after scrolling begins |
| Fade-in duration | 200ms |
| Pages | Article pages only (`/guides/[slug]`) |
| Implementation | `"use client"` component using `useScroll` from `motion/react` |
| Scroll range | Tracks `<article>` element scroll progress, not full page |
| Reduced motion | Still visible (it is informational, not decorative), but no opacity fade -- always visible at full opacity |

**Source:** CONTEXT.md D-10; CONTEXT.md specifics (burnt-orange, fades in after scrolling)

### Prefers-Reduced-Motion (ANIM-07)

| Property | Value |
|----------|-------|
| Detection | CSS `@media (prefers-reduced-motion: reduce)` AND motion's `useReducedMotion()` hook |
| Page transitions | Duration set to 0 (instant swap) |
| Scroll reveals | Opacity-only fade (no y translation), duration halved to 200ms |
| Card hover | Shadow transition only, no scale/translate |
| Button press | No scale effect |
| Nav link underline | Instant appearance, no slide animation |
| Progress bar | Always visible (informational), no opacity transition |
| FAQ accordion | CSS transition-duration set to 0.01s |

Global CSS addition:
```css
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01s !important;
    transition-duration: 0.01s !important;
  }
}
```

**Source:** CONTEXT.md D-03; REQUIREMENTS.md ANIM-07

## Performance Contract

| Metric | Target | Source |
|--------|--------|--------|
| Lighthouse Mobile Score | 90+ | CONTEXT.md D-09 |
| LCP | < 2.5s | REQUIREMENTS.md PERF-01 |
| INP | < 200ms | REQUIREMENTS.md PERF-01 |
| CLS | < 0.1 | REQUIREMENTS.md PERF-01 |
| Motion JS bundle | < 15KB gzipped (domAnimation only) | CONTEXT.md D-08 |

### Bundle Optimization Rules

1. **LazyMotion with `domAnimation`** -- NOT `domMax`. The `domMax` bundle includes gesture recognition and layout animations not needed here.
2. **Dynamic import** the features module so it code-splits from the main bundle.
3. **No `AnimatePresence`** for page transitions if it significantly increases bundle -- use CSS opacity transition with motion's `animate` prop instead.
4. **Card/button/nav hover interactions use CSS only** -- no motion library for simple hover states. Keeps these components as server components where possible.
5. **`m` components (not `motion`)** when inside LazyMotion -- `m.div` is tree-shakeable, `motion.div` is not.

### SSG Verification (PERF-02)

All content pages must remain statically generated. Verify `generateStaticParams` still works after adding motion wrappers. Motion components are client-side only -- they do not affect SSG.

### Image Optimization (PERF-03)

No article hero images exist currently. Add `next/image` import pattern in article template for future use but no immediate optimization work.

### Font Optimization (PERF-04)

Already complete from Phase 1 (next/font with `display: swap`). No changes.

**Source:** REQUIREMENTS.md PERF-01 through PERF-04; CONTEXT.md D-05 through D-09
