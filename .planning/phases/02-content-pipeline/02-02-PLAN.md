Start with the Guide type export:

**isPublished filter (CONT-03, D-10):** Every public-facing function MUST filter through `isPublished`. Use string comparison to avoid timezone issues:
  function isPublished(guide: Guide): boolean {
    return guide.publishDate <= new Date().toISOString().slice(0, 10);
  }

Note: Velite's `s.isodate()` normalizes dates. Compare as `YYYY-MM-DD` strings to ensure UTC-safe behavior (Pitfall 3 from research). The `.slice(0, 10)` extracts the date portion only.

**Functions to implement:**

1. `getAllGuides(): Guide[]` -- filter isPublished, sort by publishDate descending (newest first). Sort comparison: `new Date(b.publishDate).getTime() - new Date(a.publishDate).getTime()`
2. `getGuideBySlug(slug: string): Guide | undefined` -- find by slug using `guides.find(g => g.slug === slug)`, then return undefined if not isPublished (D-10: future articles hidden completely)
3. `getGuidesByCategory(category: string): Guide[]` -- filter by `g.category === category` AND isPublished, sort newest first
4. `getCategories(): string[]` -- return static array of all 8 category slugs: 'getting-started', 'business-structures', 'tax-obligations', 'accounting-basics', 'choosing-an-accountant', 'costs-and-fees', 'compliance-and-deadlines', 'industry-guides'
5. `getPillarGuides(): Guide[]` -- `getAllGuides().filter(g => g.isPillar)`
6. `getFeaturedGuides(): Guide[]` -- `getAllGuides().filter(g => g.featured)`
7. `getReadingTime(guide: Guide): number` -- `Math.ceil(guide.metadata.wordCount / 200)` -- 200 wpm per D-06, CONT-07. Do NOT use `guide.metadata.readingTime` directly as Velite's WPM rate may differ.

Export the `Guide` type and all 7 functions as named exports. No default exports.
